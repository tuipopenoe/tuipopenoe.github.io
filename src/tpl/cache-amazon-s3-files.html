Cache Amazon AWS S3 Files

To increase your Google PageSpeed Insights Score, one of the recommendations is to implement browser caching for your local files. Google recommends using Expires versus a Cache-Control: max-age, because it is supported by more browsers. For more recommendations on browser caching, check their recommendations.

  1. To implement Browser Caching through the AWS console, open up the console to the S3 control page.
  2. Click on the bucket that contains the assets you wish to enable caching for, and click on the properties tab.
  3. In the properties tab, under Permissions for the Bucket, click the Add/Edit CORS configuration button.
  4. Set the Cross-Origin Resource Sharing configuration to allow GET requests, with a max age in seconds of the time you would like your assets to be cached.
  5. To set the MaxAgeSeconds for 10 days, use the following configuration: 
  6.             
                
                    
                    *
                    
                    
                    GET
                    
                    
                    864000
                    
                    
                    Authorization
  7. Save the configuration properties, and check that the assets are being cached.

Enabling caching in this way can reduce the number of requests charged to your Amazon AWS S3 bucket, and decrease the time it takes to load your page.
